# Python-Library-System-v2.0-with-pickle-
Python Library v2.0 с использованием pickle вместо json и чтением бинарных файлов.

что изменилось по сравнению с json-версией

класс storage полностью переписан под pickle:

    метод save открывает файл в бинарном режиме 'wb' и использует pickle.dump()

    метод load открывает файл в бинарном режиме 'rb' и использует pickle.load()

    расширения файлов изменены с .txt на .pkl

методы load_data и save_data упрощены:

    больше не нужно преобразовывать объекты в словари через to_dict()

    не нужно восстанавливать объекты из словарей через from_dict()

    объекты book, user, librarian сохраняются и загружаются целиком

файлы, которые создаются приложением:

    books.pkl — список объектов book

    users.pkl — список объектов user

    librarians.pkl — список объектов librarian

demo_pickle.py — демонстрационный скрипт

зачем нужен второй код

показать пользователю, что данные действительно хранятся в бинарных файлах. при открытии .pkl в обычном редакторе виден нечитаемый набор байт. скрипт демонстрирует:

    открытие файла в бинарном режиме 'rb'

    вывод первых 50 байт в нечитаемом виде

    восстановление объектов через pickle.load()

    размер файла в байтах

как работает демонстрация

скрипт импортирует классы из library.py, читает файлы books.pkl, users.pkl, librarians.pkl и выводит информацию о каждом. если файлов нет, сообщает об этом.
